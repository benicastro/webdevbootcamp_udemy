The concept of HTML frames is deprecated for HTML5.

HTML frames are HTML elements that allow us to divide the screen in separate independent parts.

HTML frames can be created with < frameset > and < frame > HTML elements.

<frameset rows="50%,50%">
. <frame name="frame1" src="Frame1.html"/>
. <frame name="frame2" src="Frame2.html"/>
</frameset rows="50%,50%">

Frameset was used before instead of the body element. It acts as a container for the frames to be created in the webpage.

The important attributes for the frameset element include cols and rows that represent the number of columns or rows of the screen division, respectively, while for the frame element, we have src that represents the document that will be loaded, and name which represents the unique name for that frame.

Why we should avoid frames?

  • Frames are deprecated from HTML5.
  • Frames are acting weird on small size devices.
  • Some browsers do not support HTML frames.
  • For some browsers, the back button doesn't work as it should.

Frames failure can be covered with < noframes > HTML element where we can place anything that we want to display in cases when frames are not working for any reason.
